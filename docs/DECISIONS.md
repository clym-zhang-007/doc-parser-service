# Architecture Decisions Log

> 用于记录“为什么做/不做”，避免后续讨论反复与目标丢失。

## ADR-001：V1 先用 LlamaIndex，后续按需补 LangChain
- 日期：2026-04-03
- 决策：V1 仅接入 LlamaIndex 作为文档解析主链，LangChain 暂不实现。
- 原因：当前目标是快速跑通文档解析 API 核心链路，避免过度工程化。
- 影响：
  - 正向：实现速度快、架构简单、风险低
  - 代价：复杂 Agent/工具编排能力延后
- 结论：保持不变，V1.5 再评估引入。

## ADR-002：V1 使用异步任务模型
- 日期：2026-04-03
- 决策：采用 `FastAPI + Celery + Redis` 异步执行解析任务。
- 原因：文档解析耗时不稳定，同步接口容易超时且影响可用性。
- 影响：
  - 正向：接口稳定、可扩展
  - 代价：增加队列与 worker 组件
- 结论：纳入 V1。

## ADR-003：V1 文件存储先本地目录
- 日期：2026-04-03
- 决策：文件先落地到 `storage/`，后续再切对象存储。
- 原因：先保证功能闭环，减少外部依赖。
- 影响：
  - 正向：启动快
  - 代价：不适合多实例共享
- 结论：V1 可接受，V2 切 MinIO/S3。

## ADR-004：表结构用 Alembic，行数据用 ORM commit
- 日期：2026-04-04
- 决策：数据库 **DDL（建表、加列、索引）** 通过 Alembic 版本化迁移；**业务字段的增删改（行级）** 在应用内用 SQLAlchemy Session `commit()`，不为此重复执行 Alembic CLI。
- 原因：迁移与运行时数据职责分离，避免环境漂移；符合常见生产实践。
- 影响：
  - 正向：结构变更有审计链（`alembic/versions/`），与 `app/core/models.py` 可对照维护
  - 代价：改模型后需记得生成并审阅迁移文件
- 结论：纳入 V1 工程约定。

## ADR-005：解析链 LlamaIndex 优先、按格式回退
- 日期：2026-04-05
- 决策：Worker 内优先用 `llama_index.core.SimpleDirectoryReader` 加载单文件；若异常或文本为空，再按 `file_type` 回退到 `pypdf` / `python-docx` / 纯文本读取。
- 原因：先保证四格式在 Docker 内可稳定出字，再迭代纯 LlamaIndex 与块结构。
- 影响：
  - 正向：上线可演示端到端；复杂 PDF 仍可能依赖回退
  - 代价：`meta.parser` 会体现实际使用的读取路径
- 结论：纳入 V1 初版；后续可按格式收敛为单一引擎。
